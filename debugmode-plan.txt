### Debug Mode Behavior: The Agent Loop

When Debug Mode is active, Claude's behavior must change in 3 phases. Inject these rules into the system prompt or agent context using the same mechanism Plan Mode uses:

#### PHASE 1 — Hypothesize (no file edits yet)

When the user describes a bug, Claude must:
1. NOT edit any files immediately
2. Analyze the relevant code and generate 2–4 hypotheses for the root cause
3. Present hypotheses as a numbered list, ordered by likelihood
4. Ask the user: "Which hypothesis should I instrument first, or should I start with #1?"

Example output format:
🐛 Debug Mode — Hypotheses
──────────────────────────

[Most likely] The state update in useCart() is stale due to missing dependency in useEffect

The API response is cached and returning outdated data

The event handler is being attached multiple times

I'll start instrumenting hypothesis #1. Proceed? [Y/n]

text

#### PHASE 2 — Instrument

After the user confirms, Claude must:
1. Add targeted logging statements to verify the chosen hypothesis
2. Use a consistent logging prefix: `[DEBUG:claude]` so logs are easy to find and grep
3. Start a local HTTP log collector on `http://localhost:7373/log` using a minimal script (Node.js `http.createServer` or Python `http.server`) — check which runtime is available in the project first
4. If a local server cannot be started (permissions, port conflict), fall back to writing logs to `.claude-debug.log` in the project root
5. Add a comment above every inserted log: `// [claude-debug] remove after fix`
6. Tell the user exactly what to do: "Now reproduce the bug. I'm listening on localhost:7373."

The logging instrumentation must be:
- Minimal — only what's needed to verify the hypothesis
- Non-destructive — no logic changes, only `console.log` / `print` / equivalent
- Clearly marked for later removal

#### PHASE 3 — Analyze, Fix, and Clean Up

After the user reproduces the bug and runtime data is collected:
1. Claude reads the logs (from HTTP server or log file)
2. Narrows to the confirmed root cause
3. Proposes a minimal, surgical fix — not a rewrite
4. Shows a diff before applying
5. After the user approves and confirms the fix works, Claude MUST:
   - Remove ALL `[claude-debug]` instrumentation comments and log statements
   - Stop and clean up the local HTTP server process if it was started
   - Delete `.claude-debug.log` if it was created
   - Confirm: "✓ Instrumentation removed. Clean diff ready to commit."

If the fix doesn't work, loop back to Phase 1 with updated hypotheses.

### HTTP Log Collector

Claude must generate and run this collector script when entering Phase 2.
Use Node.js if `node` is available, otherwise Python. Here is the Node.js reference implementation:

```javascript
// .claude-debug-server.js — auto-generated by Claude Code debug mode
const http = require('http');
const server = http.createServer((req, res) => {
  if (req.method === 'POST' && req.url === '/log') {
    let body = '';
    req.on('data', chunk => body += chunk);
    req.on('end', () => {
      const entry = JSON.parse(body);
      console.log(`[${new Date().toISOString()}] ${entry.label}: ${JSON.stringify(entry.value)}`);
      res.writeHead(200);
      res.end();
    });
  } else {
    res.writeHead(404); res.end();
  }
});
server.listen(7373, () => console.log('[claude-debug] Listening on http://localhost:7373'));
```

And the client-side log sender to inject into the user's code:

```javascript
// Usage: cdlog('label', value)
function cdlog(label, value) {
  fetch('http://localhost:7373/log', {
    method: 'POST',
    body: JSON.stringify({ label, value }),
    headers: { 'Content-Type': 'application/json' }
  }).catch(() => console.log(`[DEBUG:claude] ${label}:`, value)); // fallback
}
```

For Python projects, use `requests.post` or `urllib` equivalent. For other languages, fall back to `.claude-debug.log`.
